# Corporate Wedge Generator -- 90-Day Proof Engine

The Corporate Wedge Generator is a strategic framework that forces complaint-first thinking, measurable KPIs, and kill conditions into every corporate engagement. It converts training into tangible operational outcomes before attempting to sell a system.

The core principle: **You do not sell systems. You sell solved complaints with proof.**

## Master Prompt Structure

The framework is designed to think like a COO under pressure, a CFO guarding budget, a Head of Digital drowning in failed pilots, and a board member demanding ROI -- all at once.

### Strategic Question

> "Which buyer complaint can we solve with measurable proof in under 90 days?"

### What We Provide

- Corporate AI & execution training programs
- Structured micro-team execution models
- Enterprise licensing of the execution system
- Outcome-based implementation support

### What We Avoid

- Thought leadership language
- Transformation rhetoric
- Innovation buzzwords
- Generic upskilling claims
- Cultural change narratives

### What We Want

- One clearly defined operational complaint
- That already has budget
- That a COO / CFO would admit in private
- That can be solved in 90 days or less
- With measurable financial or operational ROI
- With defined stop conditions if no traction

## Offer Architecture

### Step 1: Identify Buyer Complaints

Generate real complaints expressed in natural language by:

- COO
- CFO
- Head of Operations
- Head of Digital Transformation
- Head of L&D
- Business Unit Head

Each complaint must be specific, reference operational friction, imply budget authority, and reflect real economic pain.

**Example tone:** "We are spending money on AI tools and nothing is actually improving."

### Step 2: Convert Complaints Into 90-Day Proof Problems

For each complaint, define:

| Element | Description |
|---------|-------------|
| **Complaint** | Verbatim buyer language |
| **Bottleneck** | The operational friction point |
| **Metric** | The measurable KPI |
| **Baseline** | Current state |
| **90-Day Target** | Improvement goal |
| **Financial Impact** | Dollar/currency translation |

### Step 3: Design 90-Day Execution Offers

For each validated complaint, create a corporate offer:

| Element | Description |
|---------|-------------|
| **Title** | Clear, operational name |
| **Who It's For** | Buyer persona |
| **What We Do** | Specific intervention |
| **90-Day Plan** | Phased: Week 1-2, 3-6, 7-12 |
| **Deliverables** | Tangible outputs |
| **Measurable Outcomes** | KPI targets |
| **ROI Model** | Financial translation |
| **Kill Condition** | When we stop if no traction |
| **Pricing Structure** | Clear pricing |
| **Risk Reversal** | Buyer protection statement |

**Core sales statement structure:**

> "We deploy a small execution team that fixes one operational bottleneck in 60-90 days with measurable ROI. If it does not move the needle, we stop."

### Step 4: Separate Training vs Licensing

**Version A -- Corporate Training Program:**
- Focus on skill adoption + live business project
- Tie to measurable workflow improvement
- Position as capability upgrade

**Version B -- Enterprise Licensing:**
- Focus on installing structured execution model
- Include governance layer
- Include accountability structure
- Include recurring license fee model

### Step 5: Stress-Test the Offer

Simulate objections from CFO, COO, Procurement, and internal skeptics. Provide disciplined responses with no defensive language, no marketing tone, no exaggeration.

### Step 6: Rank Offers

Rank by speed to revenue, ease of sale, proof strength, repeatability, and margin potential.

## Tightening Constraints

### Budget Validation

For each complaint, identify the existing budget line item that currently absorbs this cost (overtime, contractor spend, software waste, rework, write-offs). If no clear budget line exists, discard the complaint.

### Magnitude Filter

Discard any complaint where 90-day impact is less than 5x the cost of the intervention.

### Kill Trigger Window

If leading indicator does not move by Week 4, intervention is paused. This behaves like capital allocation, not training.

### Political Friction Layer

Identify the internal blocker role most likely to resist this intervention and define a pre-emptive containment strategy. Answer: "Who loses control if this works?"

### Anti-Consulting Filter

Intervention must not depend on cross-functional steering committees or enterprise-wide change management. If the wedge needs a steering committee, it is dead.

## Detailed Wedge Examples

### Wedge 1: Manufacturing -- Production Stability Sprint

| Element | Detail |
|---------|--------|
| **Complaint** | "We are always expediting shipments." / "We are missing delivery dates because our planning is reactive." |
| **Budget Line** | Expediting cost, overtime, penalty clauses |
| **Executive Sponsor** | COO |
| **Scope** | One high-volume production line |
| **Team** | 4 people: Finance controller, Supply chain manager, Demand planner, AI execution coach |

**Timeline:**

| Phase | Activities |
|-------|-----------|
| **Week 1-2** | Baseline OTIF, expediting spend; Map daily planning breakdown; Select 1 high-volume line |
| **Week 3-6** | Train line supervisors in AI-assisted production forecasting; Install 15-minute daily constraint review ritual; Track variance in real time |
| **Week 7-12** | Lock execution cadence; Reduce firefighting; Eliminate expediting root causes |

| Metric | Value |
|--------|-------|
| **Primary KPI** | OTIF %, Expediting spend, Overtime hours |
| **Baseline** | OTIF 82%, Expediting 40L/quarter |
| **90-Day Target** | OTIF 92%, Expediting down 30% |
| **Financial Impact** | 12L quarterly savings, Margin uplift 1.5-2% |
| **Kill Trigger** | If OTIF does not improve 3% by Week 4, stop |

**Training Version:** Upskill 20 supervisors while fixing one live production line. Certified Execution Team tied to measurable line improvement.

**Licensing Version:** Roll out execution model across plants, quarterly governance review, performance dashboard license, recurring annual fee.

## Strongest Global Wedges

The strongest global wedges are:

- **Cash release** (Inventory, DSO)
- **Margin recovery** (Write-offs, scrap, overtime)
- **Throughput expansion** (Downtime, approvals)
- **Capex avoidance** (Utilization, OEE)

These get funded fastest. If you try to sell "productivity improvement," you drown. If you sell "cash released in 90 days," doors open.

## Strategic Pattern

| Layer | Function |
|-------|----------|
| **Training** | Bait (entry point) |
| **Proof** | Product (the 90-day result) |
| **Licensing** | Scale (recurring revenue) |

Training becomes the vehicle. Proof becomes the sale. Licensing becomes the scale layer.
