# AINEFF Governance Roles & Hierarchy

This document defines the canonical human and AI job roles required to define, validate, review, govern, and evolve the hierarchical ontology of AINEFF. This is not an org chart for operations -- it is the review, stewardship, and canonicalization workforce for AINEFF itself.

## AINEFF Hierarchy

```
AINEFF → AINEF → AINEG → AINE → AINEO → AINEOU → AINEOUT → AINEOUTM → AINEOUTMJ → AINEOUTMJS → AINEOUTMJSPR / AINEOUTMJSCYR / AINEOUTMJSCTR
```

---

## 1. Meta-Canonical Layer (AINEFF-level)

**Purpose:** Define reality, boundaries, and invariants of the entire system.
**Scope:** Ontology, epistemology, axiology, taxonomy.

### Job Roles

| Role | Responsibilities |
|------|-----------------|
| **Chief Ontologist** (Human) | Defines what exists in AINEFF and what is forbidden. Guards against personhood creep, AGI mythology, anthropomorphism. Final authority on entity definitions. |
| **Chief Epistemologist** (Human) | Defines what counts as "knowledge" vs inference vs assumption. Sets rules for confidence, decay, provenance, contestability. Owns time-validity and uncertainty semantics. |
| **Chief Taxonomist / Systems Classifier** | Owns naming, hierarchy, and composability rules. Prevents taxonomy drift, synonym explosion, and role inflation. Ensures every object exists for function, not metaphor. |
| **Chief Axiologist / Ethics Architect** | Encodes non-negotiable values into system constraints. Owns optionality preservation, harm minimization, dignity rules. Reviews every new role class for value alignment. |

These four roles must remain human-led. No delegation. Ever.

## 2. Factory Governance Layer (AINEF-level)

**Purpose:** Translate philosophy into enforceable architecture.

| Role | Responsibilities |
|------|-----------------|
| **AINEF Chief Architect** | Owns the end-to-end hierarchy from AINEF to AINE. Ensures each layer has clear responsibilities and kill boundaries. Prevents leakage of authority across layers. |
| **Governance Systems Engineer** | Translates ontology + axiology into executable governance rules. Defines policy enforcement points across the hierarchy. Works closely with courts, auditors, regulators. |
| **Lifecycle & Exit Architect** | Reviews every layer for birth, evolution, and death semantics. Ensures exit logic exists even at skill/primitive level. Prevents immortal roles, agents, or enterprises. |

## 3. Group & Enterprise Layer (AINEG / AINE)

**Purpose:** Ensure enterprises remain governable organisms, not blobs.

| Role | Responsibilities |

| **AINE Enterprise Architect** | Maps enterprise structure to organization units to teams. Ensures no role has ambiguous authority. Reviews alignment with legal entity boundaries. |
| **AINE Legal & Economic Reviewer** | Ensures each AINE maps cleanly to law, liability, and economics. Reviews whether an AINE can be sued, audited, shut down. Rejects "clever" designs that dodge accountability. |

## AINEFF Canonical Hierarchy with Kill Conditions

Each layer answers five mandatory questions:

1. Who creates me?
2. What authority do I have?
3. What I am explicitly forbidden to do?
4. Who can suspend / kill me?
5. What is destroyed vs archived on death?

If any layer cannot answer all five, it is invalid.

### AINEFF -- Framework (Non-Executable)

| Element | Detail |
|---------|--------|
| **Creator** | Human constitutional body (multilateral / standards-based) |
| **Authority** | Definition only. Zero execution. Zero enforcement. |
| **Forbidden** | Running code, owning assets, issuing commands, persisting runtime state |
| **Kill Conditions** | Superseded by a new ratified version; Revoked by constitutional authority |
| **On Death** | Old versions archived. No state transfer (because none exists). |

AINEFF survives only as text + specification.

### AINEF -- Factory

| Element | Detail |
|---------|--------|

| **Authority** | Instantiate AINEs; Validate pre-birth constraints |
| **Forbidden** | Post-birth control; Runtime intervention; Cross-AINE coordination |
| **Kill Conditions** | Regulatory breach; Retention of authority after handoff; Failure rate exceeds threshold |
| **On Death** | All in-flight instantiations halted; Existing AINEs unaffected; Factory keys destroyed |

Authority is one-way and temporary.

### AINEG -- Enterprise Group (Coordination Only)

| Element | Detail |
