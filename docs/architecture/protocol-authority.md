# Protocol Authority & Coordination Systems

Andrew Franklin Leo is a **Protocol Architect for Coordination Under Uncertainty**. This is not a brand. It is a functional role: designing coordination protocols that allow institutions, capital, humans, and machines to interact without requiring trust, ownership, or centralized authority -- and without collapsing under failure.

Frankmax is the **protocol carrier** -- the runtime environment where that coordination grammar executes.

---

## The Core Truth

A neutral coordination protocol does **not**:
- Make decisions
- Execute actions
- Enforce outcomes
- Own assets
- Represent interests

It **only**: records, routes, constrains, timestamps, verifies, expires, and exposes.

If it ever acts, it is corrupted.

Frankmax is not a company in the traditional sense. It is a **coordination substrate**.

---

## The 15 Foundational Systems

The architecture below defines how technology and AI serve as the neutral coordination standard across all parties and all systems. Think of it as a gravity well for systems, not a command center.

### 1. System of Record

**What it does:** Immutable memory.

| Layer | Description |
|---|---|
| **Tech layer** | Cryptographic ledgers (audit-chain, not hype-chain), event-sourcing databases, hash-linked documents |
| **AI role** | Zero intelligence. Validates structure, not meaning. Flags inconsistencies, never "fixes" them |

Whoever controls the record controls history. Frankmax does not control -- it freezes.

---

### 2. System of Execution

**What it does:** Real-world actions.

| Layer | Description |
|---|---|
| **Tech layer** | External systems (banks, courts, ops teams, APIs), smart workflows outside the protocol |
| **AI role** | Pre-flight simulation only. No actuation rights. Mandatory human ratification |

**Boundary:** Frankmax never executes. Execution touching it is a violation.

---

### 3. System of Coordination (Frankmax)

**What it does:** Routes intent under constraint.

| Layer | Description |
|---|---|
| **Tech layer** | Role-based routing engines, conflict-of-interest detection, time-delayed handoffs |
| **AI role** | Pattern detection, deadlock detection, escalation triggers |

This is the heart. Frankmax does not decide what happens. It decides **who must talk to whom, when, and under what constraints**.

## The Protocol's Core Properties

This architecture works because:

| Property | Mechanism |
|---|---|
| **AI has no authority** | AI records, routes, and flags -- never decides |
| **Humans have friction** | Mandatory delays, ratification points, cooling periods |
| **Power has boundaries** | Every role has a ceiling, every authority has a scope |
| **Memory is immutable** | Records cannot be altered after the fact |
| **Systems decay** | Nothing persists without renewal |

## Protocol Architecture Principles

### Coordinate Without Owning

The protocol defines:

1. **Roles, not people** -- Bind to functions (Originator, Operator, Verifier, Capital Provider, Beneficiary, Arbiter). Anyone can occupy a role if they meet criteria. Anyone can be replaced.

2. **Interfaces, not workflows** -- Define inputs, outputs, and failure states. How work is achieved is the participant's problem.

3. **Default rules, not promises** -- Define what happens when things go wrong. Defaults are monetizable. People pay to avoid them.

4. **External execution and capital** -- Capital never comes from the protocol. Labor never reports to the protocol. Assets are never on the protocol's balance sheet.

5. **Legible outcomes, not optimal ones** -- Make results measurable and comparable. Legibility creates pressure without authority.

### The Litmus Test

Ask: "If this fails publicly, can Frankmax say 'the protocol behaved as designed' and walk away clean?"

If not, the protocol is still owning something. Fix that before scaling.

Frankmax is a **neutral coordination infrastructure** that lets complex actors work together without collapsing into ownership, liability, or trust failure.

Frankmax charges for:

| Function | Mechanism |
|---|---|
| **Admission** | Qualification and onboarding fees |
| **Routing** | Matching, placement, priority routing |
| **Verification** | Proof, audits, checks |

None of these require ownership. Frankmax charges for **reduced uncertainty**.
