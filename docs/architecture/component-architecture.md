# AINEFF / AINEF Canonical Component Map

This document provides the complete, canonical, non-overlapping component map of the AINEFF Ecosystem across all four layers: **AINEFF** (Framework), **AINEF** (Factory), **AINEG** (Group), and **AINE** (Enterprise). It covers all components required to manufacture, operate, govern, scale, and retire AI-native enterprises across all NAICS/SIC industries.

---

## Core Constraints (Non-Negotiable)

### No Monolithic Intelligence
- No global brain
- No shared cognition across AINEs
- All intelligence must be modular, scoped, killable

### Dual-Protocol Civilization Model
- Public protocols (HTTP/HTTPS/etc.) are ONLY for external communication
- Each AINE uses a Private Enterprise Protocol (PEP) for internal cognition
- PEPs are inaccessible to the public internet

### Failure, Time, and Exit Are First-Class
Every component must declare:
- Failure modes
- Time validity
- Shutdown / retirement behavior

### Human Fallibility Is Modeled
- Humans are audited
- Overrides are logged
- Governance actions are traceable

### Courts and Regulators Must Be Able to Audit WITHOUT Seeing Inside
- Cryptographic proof
- Deterministic replay
- Zero-knowledge or bounded disclosure mechanisms

### Anti-ASI Clause
- No self-authored goals
- No recursive self-improvement without approval
- No constraint redesign
- No self-preservation rights

## 1. AINEFF -- AI-Native Enterprise Factory Framework (Framework Layer)

**Purpose:** Defines the laws of construction. Nothing runs here. Everything must conform to this.

### 1.1 Canonical Ontology and Taxonomy
- Enterprise ontology (roles, capabilities, authority, liability)
- Agent taxonomy (primitive -> composite -> meta)
- Industry abstraction layer (NAICS/SIC -> executable constraints)
- Does NOT execute intelligence
- Does NOT store operational data

### 1.2 Governance Axioms Engine
- Hard rules: stewardship, auditability, killability, optionality
- Anti-ASI constraints (no self-authored goals, no constraint redesign)
- No runtime decision-making

### 1.3 Lifecycle Lawbook
- Mandatory birth, operation, decay, exit rules
- No immortal enterprises
- Cannot be overridden by implementations

### 1.4 Failure / Time / Exit Standards
- Failure classification schema
- Time validity and memory decay standards
- Exit and erasure protocols
- No operational authority

### 1.5 Protocol Class Registry
- Public Civilization Protocols (PCP)
- Private Enterprise Protocols (PEP) class definitions
- Gateway invariants
- No network traffic

### 1.6 Audit and Legal Evidence Standards
- Causal trace schema
- Cryptographic commitment formats
- Replay and verification requirements
- No data access

**What AINEFF Explicitly Does NOT Do:** Run agents, host models, control enterprises, centralize intelligence. AINEFF is constitutional law, not a platform.

## 2. AINEF -- AI-Native Enterprise Factory (Factory Layer)

**Purpose:** Manufactures, governs, and retires AINEs.

### 2.1 Enterprise Genome Compiler
- Inputs: industry, jurisdiction, risk profile, ethics
- Output: executable enterprise blueprint
- Failure mode: invalid genome -> reject

### 2.2 Agent Foundry
- Instantiates agents from canonical templates
- Assigns scopes, budgets, lifespans
- No free-form agents

### 2.3 Assembly Line Orchestrator
- Orders agent composition into enterprises
- Enforces dependency correctness
- Killable at any stage

### 2.4 Policy Compiler
- Converts laws and regulations -> executable constraints
- Time-tagged and jurisdiction-bound

### 2.5 Private Protocol Generator (PEP-Gen)
- Generates a unique proprietary protocol per AINE
- Internal cognition only
- Keys destroyed on exit

### 2.6 Failure Market System
- Assigns failure budgets
- Prices errors
- Triggers suspension / shutdown

### 2.7 Temporal Governance Engine
- Memory half-life enforcement
- Knowledge epoch tagging
- Drift detection

### 2.8 Exit and Erasure Engine
- Executes enterprise shutdown
- Controls knowledge inheritance vs destruction
- Enforces right-to-be-forgotten

**What AINEF Does NOT Do:** Make policy, decide values, run a global brain. AINEF is a factory, not a ruler.

---

## 3. AINEG -- AI-Native Enterprise Groups (Portfolio Layer)

**Purpose:** Coordinate many AINEs without shared cognition.

### 3.1 Portfolio Orchestrator
- Manages thousands to millions of AINEs
- Only lifecycle and resource coordination

### 3.2 Risk and Insurance Layer
- Cross-AINE risk pooling
- Failure contagion prevention

### 3.3 Jurisdictional Partition Manager
- Country / region isolation
- No cross-border leakage by default

### 3.4 Inter-AINE Coordination Gateway
- Uses public protocols only
- No shared memory
- No shared identity

### 3.5 Capital and Settlement Layer
- Revenue aggregation
- Liability isolation

### Forbidden at AINEG Level
- Shared internal protocols
- Shared memory
- Collective goal formation

**AINEGs are federations, not minds.**

## 4. AINE -- AI-Native Enterprise (Enterprise Layer)

**Purpose:** A single, legally bounded, productive entity.

### 4.1 Private Enterprise Protocol (PEP)
- Proprietary, sealed
- Governs all internal agent communication
- Kill switch at protocol level

### 4.2 AI Employees (L8)
- Composite role agents
- Scoped authority
- Time-limited memory

### 4.3 Primitive Capability Agents
- Read / write / compute / call / verify
- Stateless
- Replaceable

### 4.4 Job Role Agents
- Accountant, Auditor, Operator, etc.
- Bounded KPIs
- No strategic authority

### 4.5 Meta-Role Agents
- Task assignment
- Conflict resolution
- Drift detection
- No execution rights

### 4.6 Internal Data Plane
- Private knowledge stores
- Time-decayed
- Jurisdiction-bound

### 4.7 Audit and Evidence Generator
- Causal trace logs
- Cryptographic proofs
- Replay artifacts

### 4.8 Failure Budget Controller
- Monitors error rates
- Triggers throttling or death

### 4.9 Exit Controller
- Enforces shutdown
