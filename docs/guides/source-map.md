# Source Document Map

This page maps every documentation page in the AINEFF Ecosystem back to its original source documents. The documentation was synthesized from **38 markdown files** (1.md through 38.md) plus several PPTX, DOCX, and XLSX source files. Each documentation page draws from one or more source documents.

This map enables you to **trace any claim back to its provenance** -- verifying where a concept originated and how it was transformed into its current documentation form.

## How to Read This Map

- **Documentation Page** -- The page as it appears in the Docusaurus site
- **Source Document(s)** -- The original file(s) from which the content was synthesized
- **Key Concepts Extracted** -- The primary ideas drawn from each source into the documentation page

## Provenance Principles

1. **Every claim is traceable.** Every assertion in the documentation can be traced to at least one source document.
2. **Synthesis, not invention.** The documentation synthesizes and reorganizes source material -- it does not introduce concepts that do not exist in the sources.
3. **Source fidelity.** Where source documents provide specific numbers, dates, or thresholds, the documentation preserves them without modification.
4. **Cross-reference integrity.** When a documentation page references another page, both pages trace back to consistent source material.
5. **Version awareness.** Source documents represent a point-in-time snapshot. The documentation should be updated when sources are revised.
